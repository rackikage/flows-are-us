# FLOWS В· Business Social Hub вЂ” Specification

The one-stop app for a business's social presence, backed **entirely** by
Composio-provided APIs. This document is normative: a change that violates an
invariant below is wrong, even if it works.

---

## 1. Purpose

FLOWS lets a business run Instagram + Facebook from one place: see everything,
publish anywhere, answer everyone, and measure it вЂ” on a desktop or installed
on any phone. It is operated by non-technical people; every label, error and
hint speaks plain business language.

## 2. Invariants

### I1 вЂ” Composio clean pipe, only
Every platform interaction goes through the Composio MCP endpoint
(`composio_pipe.ComposioPipe` в†’ `COMPOSIO_MULTI_EXECUTE_TOOL`). No hand-made
Graph API calls, no imported platform SDKs, no scraping вЂ” if Composio doesn't
provide the tool, the hub doesn't do the thing. (The dead `composio-core` SDK
must never return; the pipe is HTTP to `connect.composio.dev`.)

### I2 вЂ” Use Composio's tools as designed, don't reinvent them
When a capability exists as a Composio tool, wire that tool with its exact
schema. Multi-step platform flows (IG container в†’ publish; carousel children
в†’ parent в†’ publish) follow Composio's documented plan, not improvised calls.

### I3 вЂ” Nothing goes public without a human confirming
Publishing, scheduling, replying to comments or DMs, editing and deleting all
require an explicit user action in the UI; destructive actions take a second
confirming click. The server never auto-posts.

### I4 вЂ” Secrets stay out of the browser and the repo
Composio credentials come from `~/.claude.json` or env vars. Page
`access_token` fields are scrubbed server-side (`scrub()`) before any payload
reaches the UI. Nothing secret is committed.

### I5 вЂ” Live data is never served stale to a decision
`/api/*` responses may be cached for at most 90 seconds; any write clears the
cache. The service worker never caches `/api/*`.

### I6 вЂ” Works on phones, honestly
The hub binds all interfaces, installs as a PWA on iPhone (Safari в†’ Add to
Home Screen) and Samsung/Android (Chrome в†’ Install), and every view is usable
at 375 px. The QR install card is generated by the dependency-free `qr.py` вЂ”
no QR library, no external service.

### I7 вЂ” Platform limits are explained, not hidden
Where Meta constrains the API (25 IG posts/day, FB-only scheduling, 24-hour DM
reply window, 1,000-follower insights gate, query-free JPEG URLs), the UI says
so in plain language before the user hits the wall.

`FACEBOOK_GET_POST_REACTIONS`, `FACEBOOK_GET_SCHEDULED_POSTS`

**Facebook вЂ” write:** `FACEBOOK_CREATE_POST`, `FACEBOOK_CREATE_PHOTO_POST`,

Every slug above appears in the Integrations capability table
(`TOOL_REGISTRY` in `server.py`); the table and this matrix must stay in sync.

## 7. Identity

Deep indigo-black base (`#0a0a12`), iris accent (`#8f7aff` / `#6a4dff`),
